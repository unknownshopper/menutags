NFC Token + Dominio — Sandbox

Archivo aislado (HTML/CSS/JS) para entender por qué un flujo de token por NFC funciona en subdominios propios pero falla en GitHub Pages.

1) El punto clave

Si el menú está en GitHub Pages (ej. unknownshopper.github.io) y el gateway/token está en unknownshoppers.com, entonces:

- Las cookies con Domain=.unknownshoppers.com NO se comparten con github.io.

Resultado: aunque el NFC pase por go.unknownshoppers.com y se cree el token, el menú en GitHub Pages no lo puede “ver”.

Solución real: que el menú viva en subdominios.unknownshoppers.com.

2) Flujo recomendado (un solo uso)

Este es el flujo que sí te da “NFC-only” de verdad:

1) NFC abre:
   https://go.unknownshoppers.com/t/SZ-001

2) Backend valida tag_id, crea token (expira rápido) y set-cookie:
   mt_token=...; Domain=.unknownshoppers.com; Secure; HttpOnly; SameSite=Lax

3) Backend redirige a:
   https://srchorizo.unknownshoppers.com/

4) El menú (JS) llama:
   https://go.unknownshoppers.com/api/verify?menu=srchorizo

5) verify marca el token como used=1 (un solo uso) y responde ok.

3) ¿Por qué “funciona en uno y en otro no”?

Funciona cuando:

Gateway: go.unknownshoppers.com y Menú: srchorizo.unknownshoppers.com (mismo “site” base: unknownshoppers.com)

Falla cuando:

Gateway: go.unknownshoppers.com pero Menú: unknownshopper.github.io (otro dominio base, cookies no viajan)

Además de cookies, suelen pegar estos puntos:

- CORS: el menú hace fetch a go.unknownshoppers.com; el backend debe permitirlo.

- credentials: si dependes de cookie, el fetch debe llevar credentials: "include".

- HTTPS: si usas Secure en cookie (debes), necesitas HTTPS real en subdominios.

4) Checklist backend (cPanel/PHP)

Cookies
- Set-Cookie: mt_token=...;
  Domain=.unknownshoppers.com;
  Path=/;
  Secure;
  HttpOnly;
  SameSite=Lax

Verify endpoint
- Valida token
- Expira rápido (2 min recomendado)
- Un solo uso: used=1 al primer verify OK

CORS (si el menú y go están en diferentes subdominios)
- Access-Control-Allow-Origin: https://srchorizo.unknownshoppers.com
- Access-Control-Allow-Credentials: true

Headers
- Content-Type: application/json

5) Snippet JS (lo que pegarías en cada menú)

Esto NO “protege” por sí solo; protege porque el backend controla token/cookie.

// En https://srchorizo.unknownshoppers.com/
(async function gate() {
  try {
    const res = await fetch(
      'https://go.unknownshoppers.com/api/verify?menu=srchorizo',
      { credentials: 'include' }
    );
    if (!res.ok) throw new Error('not authorized');
  } catch (e) {
    document.body.innerHTML =
      '<div style="padding:24px;font-family:system-ui">' +
      '<h2>Acceso requerido</h2>' +
      '<p>Acerca tu teléfono al sticker NFC para abrir el menú.</p>' +
      '</div>';
  }
})();
